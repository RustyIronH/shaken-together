  <name>Task 1: Multi-touch drag system with fling-on-release</name>
  <files>src/input/drag-manager.ts, src/input/multi-touch.ts, src/input/__tests__/drag.test.ts</files>
  <behavior>
    - handlePointerDown uses Query.point to find body under pointer; creates a constraint anchoring body to pointer position
    - Same body cannot be grabbed by two different pointers
    - handlePointerMove updates constraint pointA to follow pointer
    - handlePointerUp removes constraint and applies fling velocity to body
    - calculateReleaseVelocity averages last 5 position samples to compute velocity
    - Zero velocity returned when fewer than 2 samples
    - setupMultiTouch registers pointerdown/pointermove/pointerup/pointercancel on canvas
    - Each pointer tracked independently via pointerId
  </behavior>
  <action>
Create `src/input/drag-manager.ts`:

- `screenToWorld(clientX, clientY, canvas)`: Converts screen coordinates to canvas coordinates using `canvas.getBoundingClientRect()` and canvas width/height ratio. Critical for correct hit testing (see research pitfall #6).

- `handlePointerDown(e, engine, scene, canvas)`:
  1. Convert pointer position to world coordinates via `screenToWorld`
  2. Get all bodies with `Composite.allBodies(engine.world)`
  3. Hit test with `Query.point(bodies, point)`
  4. Filter out static bodies (walls)
  5. If hit: check no existing drag on this body (`scene.activeDrags`)
  6. Create a `Constraint` with `pointA: worldPoint`, `bodyB: hitBody`, `pointB: { x: worldX - body.position.x, y: worldY - body.position.y }`, `stiffness: 0.8`, `length: 0`, `render: { visible: false }`
  7. Add constraint to world via `Composite.add`
  8. Find which ragdoll owns this body (search scene.ragdolls for matching body)
  9. Create VelocityTracker, store ActiveDrag in `scene.activeDrags` keyed by pointerId
  10. Update z-order: move grabbed ragdoll's composite to end of world children (renders on top)

- `handlePointerMove(e, scene, canvas)`:
  1. Look up ActiveDrag by pointerId
  2. If found: update `drag.constraint.pointA` to new world position
  3. Track position sample for velocity calculation: `trackPosition(drag.velocityTracker, worldX, worldY)`

- `handlePointerUp(e, engine, scene, canvas)`:
  1. Look up ActiveDrag by pointerId
  2. Calculate release velocity via `calculateReleaseVelocity(drag.velocityTracker)`
  3. Remove constraint from world: `Composite.remove(engine.world, drag.constraint)`
  4. Apply velocity to body: `Body.setVelocity(drag.body, { x: vel.x, y: vel.y })` -- cap to MAX_VELOCITY
  5. Delete from scene.activeDrags

- `trackPosition(tracker, x, y)`: Push `{ x, y, time: performance.now() }`. Keep only last `VELOCITY_SAMPLE_COUNT` samples.

- `calculateReleaseVelocity(tracker)`: Take first and last of tracked positions. Compute `dx/dt` and `dy/dt` in physics units (multiply by `FLING_VELOCITY_SCALE`). Return `{ x: 0, y: 0 }` if fewer than 2 samples.

Create `src/input/multi-touch.ts`:

- `setupMultiTouch(canvas, engine, scene)`:
  1. Add `pointerdown` listener -> `handlePointerDown`
  2. Add `pointermove` listener -> `handlePointerMove`
  3. Add `pointerup` listener -> `handlePointerUp`
  4. Add `pointercancel` listener -> `handlePointerUp` (treat cancel as release, prevents orphaned constraints)
  5. Set `canvas.style.touchAction = 'none'` (prevents browser default gestures)
  6. Return cleanup function that removes all listeners

- `cleanupMultiTouch(cleanup)`: Calls the cleanup function returned by setup.

Fill in test stubs in `src/input/__tests__/drag.test.ts`:
- Test `calculateReleaseVelocity` with known position samples -> expected velocity
- Test `calculateReleaseVelocity` with < 2 samples -> `{ x: 0, y: 0 }`
- Test hit testing: create an engine with a body at known position, call Query.point, verify body found
- Test no double-grab: if body is already in activeDrags, handlePointerDown should not create a second constraint

<task type="auto">
  <name>Task 2: Canvas2D debug renderer with glow and z-order</name>
  <files>src/renderer/debug-renderer.ts, src/renderer/colors.ts</files>
  <action>
Create `src/renderer/colors.ts`:

- `bringToFront(engine, ragdoll)`: Moves the ragdoll composite to the end of `engine.world.composites` array so it renders last (on top). Use `Composite.move(engine.world, ragdoll.composite, engine.world.composites.length - 1)` or manually reorder.

- `drawGlow(ctx, body, color, radius)`: Draws a soft glow effect around a body:
  1. Save context
  2. Set `ctx.shadowBlur = radius` (e.g., 15)
  3. Set `ctx.shadowColor = color`
  4. Draw the body shape (circle or rect) filled with semi-transparent highlight color
  5. Restore context
  This creates a visible highlight on grabbed bodies per user decision.

Create `src/renderer/debug-renderer.ts`:

- `createDebugRenderer(canvas)`: Gets 2D context, stores canvas reference, returns renderer object.

- `renderFrame(ctx, engine, scene)`:
  1. Clear canvas: `ctx.clearRect(0, 0, canvas.width, canvas.height)`
  2. Draw gradient background using `ctx.createLinearGradient` with `BG_GRADIENT_TOP` and `BG_GRADIENT_BOTTOM`
  3. Get all bodies from engine.world via `Composite.allBodies(engine.world)`
  4. Sort bodies by their composite's position in `engine.world.composites` for z-order (bodies from composites later in the array render on top)
  5. For each non-static body:
     a. Save context, translate to `body.position`, rotate by `body.angle`
     b. Set `fillStyle` from `body.render.fillStyle` (set during ragdoll creation from colorScheme)
     c. If body.label is 'head' or starts with 'hand': draw circle with `body.circleRadius`
     d. If body.label is 'foot': draw circle with `body.circleRadius`
     e. Otherwise: draw rectangle using bounds width/height centered on origin
     f. Restore context
  6. Check if body is currently being dragged (find in `scene.activeDrags` values):
     a. If dragged: call `drawGlow(ctx, body, ragdoll.colorScheme.highlight, 15)`
  7. Draw constraints as semi-transparent white lines:
     a. For each constraint with both bodyA and bodyB
     b. Skip drag constraints (render.visible === false)
     c. Draw line from bodyA.position + pointA to bodyB.position + pointB
     d. Use `rgba(255,255,255,0.3)` stroke, lineWidth 1

- `resizeCanvas(canvas)`: Set `canvas.width = window.innerWidth`, `canvas.height = window.innerHeight`. Call on init and on window resize. This ensures canvas pixel dimensions match CSS dimensions (prevents coordinate mismatch per research pitfall #6).

The renderer should be called from requestAnimationFrame. The render loop integration happens in Plan 04 (main.ts), but the renderer itself must be complete and callable here.
