<task type="auto">
  <name>Task 3: Create RagdollSprite class with Container and 15 Graphics children</name>
  <files>src/renderer/ragdoll-sprite.ts</files>
  <action>
1. Create `src/renderer/ragdoll-sprite.ts`:
   - Export `RagdollSprite` interface:
     ```typescript
     export interface RagdollSprite {
       container: Container;           // Root container for this ragdoll
       parts: Map<string, Graphics>;   // body part label -> Graphics object
       headGraphic: Graphics;          // Quick reference for face expression swaps
       characterId: CharacterId;       // Which character skin is applied
       ragdollId: string;              // Links to RagdollInstance.id
     }
     ```
   - Export `createRagdollSprite(characterId: CharacterId, ragdollId: string): RagdollSprite`:
     - Gets `CharacterDefinition` from `getCharacter(characterId)`
     - Creates a `Container` with `label: 'ragdoll-' + ragdollId`
     - Creates 15 `Graphics` objects, one per body part, using `new Graphics(context)` where `context` comes from `character.bodyParts.get(label)`
     - Adds them to the container in draw order (back to front):
       `footL, lowerLegL, upperLegL, footR, lowerLegR, upperLegR, lowerTorso, upperTorso, handL, forearmL, upperArmL, handR, forearmR, upperArmR, head`
     - Head Graphics uses the `neutral` face expression context from `character.faceExpressions`
     - Sets each Graphics `label` to match the body part label
     - Returns the RagdollSprite

   - Export `swapCharacterSkin(sprite: RagdollSprite, newCharacterId: CharacterId): void`:
     - Gets the new CharacterDefinition
     - For each body part label in `sprite.parts`: replaces the Graphics object's `context` with the new character's GraphicsContext for that label
     - Updates `sprite.headGraphic.context` to the new character's `neutral` face
     - Updates `sprite.characterId`
     - This is an in-place swap -- Container structure unchanged, positions preserved

   - Export `destroyRagdollSprite(sprite: RagdollSprite): void`:
     - Calls `graphic.destroy()` on each Graphics in `sprite.parts`
     - Calls `sprite.container.destroy({ children: true })`
     - Does NOT destroy the shared GraphicsContexts (they belong to the character registry)

   **Key PixiJS v8 patterns:**
   - `new Graphics(context)` creates a Graphics using a shared GraphicsContext -- GPU geometry is shared
   - Only `Container` can have children (not Graphics) -- the container holds the Graphics children
   - `graphic.context = newContext` swaps the visual appearance cheaply
  </action>
