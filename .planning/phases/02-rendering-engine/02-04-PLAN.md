  <action>
Present the user with the following verification checklist. Run `npm run dev` to start the dev server and provide the URL.

**What was built:** Complete PixiJS rendering engine replacing the Canvas2D debug renderer. 4 cartoon characters with distinct silhouettes (Slim, Round, Buff, Tiny), reactive face expressions, visual effects (squash-stretch, speed lines, impact flashes), character selection UI in the slide-out panel, and responsive mobile canvas.

**How to verify:**

1. Run `npm run dev` and open the app in a browser

2. **Character appearance (RNDR-01):**
   - Verify ragdolls render as cartoon characters (not colored rectangles/circles)
   - Confirm 4 distinct character silhouettes are visible (tall/thin, short/round, buff, small)
   - Check that body parts overlap seamlessly at joints (no visible gaps or pins)
   - Check that characters have thick outlines and bold colors ("Dumb Ways to Die" style)

3. **Visual effects (RNDR-01):**
   - Drag a ragdoll and fling it hard against a wall -- verify impact flash appears at the collision point
   - Watch the ragdoll's face -- it should change to surprised/dazed expression on hard impact, then revert
   - Fling a ragdoll fast -- verify body parts stretch slightly in the direction of motion
   - Watch fast-moving hands/feet -- subtle speed lines should trail behind them

4. **Drag feedback:**
   - Grab a ragdoll and hold -- verify it scales up slightly (~5%) and appears "lifted"
   - Verify the grabbed ragdoll renders on top of other ragdolls

5. **Character selection (RNDR-02):**
   - Open the slide-out panel (hamburger menu)
   - Verify a "Characters" section shows each doll slot with a dropdown
   - Change a character via the dropdown -- verify the ragdoll's appearance changes immediately
   - Verify position and entanglement are preserved during the swap
   - Try selecting a character already used by another doll -- verify the two dolls swap characters

6. **Performance (RNDR-03):**
   - Set doll count to 5 and verify the canvas still feels smooth (60fps)
   - Open browser DevTools Performance tab and check for consistent frame timing

7. **Responsive layout (PLAT-03):**
   - Resize the browser window -- verify canvas fills the viewport
   - Use Chrome DevTools device emulation to test various phone sizes in portrait mode
   - Verify no scrollbars appear and the canvas fills the screen edge to edge

8. **Regression check:**
   - Verify multi-touch drag still works (two fingers on two different ragdolls)
   - Switch between Realistic and Goofy modes -- verify physics still behaves differently
   - Change doll count -- verify ragdolls are added/removed and character selectors update
   - Reset scene -- verify all ragdolls teleport to new positions

**Resume signal:** Type "approved" if all checks pass, or describe any issues that need fixing.
