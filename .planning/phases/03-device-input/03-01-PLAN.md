  <action>
    Create src/input/shake-manager.ts implementing the shake detection pipeline:

    1. **Module state**: A ShakeState object initialized with defaults (smoothedX/Y = 0, active = false, permissionState = 'pending').

    2. **initShake(engine, scene)** async function:
       - Check `typeof DeviceMotionEvent === 'undefined'` -> return 'unsupported'
       - Check `typeof (DeviceMotionEvent as any).requestPermission === 'function'` -> call it from within the function (the caller in main.ts will ensure this is triggered from a user gesture). Return the result ('granted'/'denied').
       - Otherwise (Android over HTTPS): add devicemotion event listener, return 'granted'
       - On 'granted': add `window.addEventListener('devicemotion', handler)` where handler calls `handleMotion`
       - Store the engine and scene references for the handler to use
       - Return the permission state

    3. **handleMotion(event: DeviceMotionEvent)** (internal, called by event listener):
       - Guard: `if (!accel || accel.x === null || accel.y === null) return`
       - Apply exponential smoothing: `smoothedX = smoothedX + alpha * (rawX - smoothedX)`
       - Calculate magnitude: `sqrt(smoothedX^2 + smoothedY^2)`
       - Determine if active: `|magnitude - 9.8| > activeThreshold`
       - Map to gravity (snow globe model):
         - `engine.gravity.x = smoothedX * modeMultiplier * gravityScaleFactor / defaultScale`
           Wait -- simpler approach per Research: device X maps to gravity X direction, device Y (inverted) maps to gravity Y direction. The accelerometer at rest gives (0, -9.8, 0) in portrait. We want rest = default gravity (0, 1). So:
         - `engine.gravity.x = smoothedX * scaleFactor * modeMultiplier`
         - `engine.gravity.y = -smoothedY * scaleFactor * modeMultiplier` (invert Y)
         - Where scaleFactor normalizes so 9.8 m/s^2 -> mode's default gravity scale
         - modeMultiplier = `scene.currentMode.name === 'goofy' ? SHAKE_CONFIG.goofyMultiplier : 1.0`
       - If magnitude delta > wakeThreshold: wake all sleeping non-static bodies using `Sleeping.set(body, false)`
       - Update lastEventTime to `performance.now()`

    4. **updateGravityLerp(engine, mode, deltaMs)** (called every frame from ticker):
       - If shake is active (recent event within ~200ms): do nothing (let handleMotion control gravity)
       - If shake is inactive (no recent event): lerp gravity back to mode defaults
       - Use frame-rate independent lerp: `t = 1 - Math.pow(1 - lerpSpeed, deltaMs / 16.67)`
       - `engine.gravity.x += (mode.gravity.x - engine.gravity.x) * t`
       - `engine.gravity.y += (mode.gravity.y - engine.gravity.y) * t`
       - `engine.gravity.scale += (mode.gravity.scale - engine.gravity.scale) * t`

    5. **getShakeState()** returns the current ShakeState (for UI decisions like showing fallback button).

    6. **Export**: `initShake`, `updateGravityLerp`, `getShakeState`, and for testing: `handleMotion`, `wakeAllBodies`, `smooth`, `_resetShakeState` (test helper following project pattern).

    Important: Use `accelerationIncludingGravity` (not `acceleration`) per Research -- works on devices without gyroscopes.
  </action>
