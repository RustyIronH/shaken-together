  <action>
Create `src/capture/screenshot.ts` exporting:

```typescript
export interface CaptureResult {
  clean: string;       // data URL, effects OFF
  withEffects: string; // data URL, effects ON
}
```

Export function `captureScreenshots(app: Application, effectsLayer: Container, scene: SceneState, spriteMap: Map<string, RagdollSprite>): CaptureResult` that:

1. Clears all active drags from `scene.activeDrags`: for each drag, call `Composite.remove(engine.world, drag.constraint)` -- but since we don't have engine reference in this function, instead accept a `cleanupDrags: () => void` callback parameter that main.ts will provide. The callback should iterate `scene.activeDrags`, remove each constraint from the world, and clear the map.
   - Actually, simpler approach: just iterate `scene.activeDrags` and clear the map. The constraints will become orphaned but Runner is stopped so they won't cause issues. The constraint references go away when drags are re-created on next pointer down.
   - Even simpler: accept the spriteMap so we can reset scales. For drag cleanup, just clear `scene.activeDrags` (the constraints are already in the world but with Runner stopped, harmless). On resume, any pointerup events will just no-op because the activeDrags map is empty.

2. Resets all sprite container scales to 1.0: iterate spriteMap values, set each `sprite.container.scale.set(1.0)`.

3. Wraps the two-pass capture in try/finally:
   - Pass 1: `effectsLayer.visible = false`, extract canvas, toDataURL
   - Pass 2: `effectsLayer.visible = true`, extract canvas, toDataURL
   - Finally: `effectsLayer.visible = true` (restore)

4. Uses `app.renderer.extract.canvas({ target: app.stage }) as HTMLCanvasElement` for each pass, then `.toDataURL('image/png')`.

5. Returns `{ clean, withEffects }`.

**Signature refinement:** Since the function needs to clear drags and reset sprites (which are concerns outside pure capture), keep it focused:

```typescript
export function captureScreenshots(
  app: Application,
  effectsLayer: Container,
): CaptureResult
```

And export a separate `prepareForCapture(scene: SceneState, spriteMap: Map<string, RagdollSprite>): void` that clears drags and resets scales. main.ts calls prepareForCapture then captureScreenshots.

- Mock effectsLayer as `{ visible: true }`.
- Verify visibility toggling and restoration.
- Mock scene with activeDrags Map and spriteMap for prepareForCapture tests.

Use the project pattern: test files in `src/{module}/__tests__/*.test.ts`.
