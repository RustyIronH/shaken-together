   to:
   ```typescript

   ```
   This is the critical fix identified in Research (Pitfall 1).

3. **Create capture button** after the shake button setup (after line ~190), always visible:
   ```typescript
   const captureBtn = createCaptureButton(() => {
     // 1. Freeze physics
     stopEngine(runner);

     // 2. Prepare: clear drags, reset sprite scales
     prepareForCapture(scene, spriteMap);

     // 3. Capture both image variants
     const images = captureScreenshots(renderer.app, renderer.effectsLayer);

     // 4. Disable capture button during preview
     disableCaptureButton(captureBtn);

     // 5. Show preview overlay with flash
     showCaptureOverlay(uiRoot, images, {
       onSave: (dataUrl: string) => {
         // Download PNG
         const link = document.createElement('a');
         link.download = `shaken-together-${Date.now()}.png`;
         link.href = dataUrl;
         document.body.appendChild(link);
         link.click();
         document.body.removeChild(link);

         // Resume physics and re-enable button
         Runner.run(runner, engine);
         enableCaptureButton(captureBtn);
       },
       onDiscard: () => {
         // Resume physics from frozen state (no reset, per locked decision)
         Runner.run(runner, engine);
         enableCaptureButton(captureBtn);
       },
     });
   });
   uiRoot.appendChild(captureBtn);
   ```

4. **Capture button is always visible** -- no conditional display. It renders for all users (motion and fallback alike), per the locked decision "Button stays on screen at all times, including during active shaking."

5. **Sequence correctness** (per Research Pitfall 7):
   - Freeze physics FIRST (so captured positions match what user saw)
   - Then capture images
   - Then show flash + overlay (cosmetic feedback)
   - The capture happens at the moment of tap, flash is just visual confirmation

6. **No changes to the ticker** -- the ticker naturally continues running (PixiJS ticker is separate from Matter.Runner). But since physics is stopped, bodies won't move. Speed lines and impact flashes will naturally decay to invisible. Squash-stretch will show the last frame's deformation (acceptable -- bodies are frozen).

**Do NOT:**
- Reset the scene on resume (locked: "Physics resumes from frozen state when preview is dismissed")
- Allow posing/dragging while frozen (locked: "No posing/dragging while frozen")
- Re-render effects at toggle time (locked: "frozen in place, not re-rendered" -- the two pre-captured images handle this)
