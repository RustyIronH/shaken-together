<feature>
  <name>Feature 1: Circular Replay Buffer</name>
  <files>src/capture/replay-buffer.ts, src/capture/__tests__/replay-buffer.test.ts</files>
  <behavior>
    - createReplayBuffer(width, height) returns a ReplayBuffer with capacity 30 (10fps * 3s), recording=true, frameCount=0
    - pushFrame(buffer, rgba) when recording=true stores frame at writeIndex, advances writeIndex, increments frameCount up to capacity
    - pushFrame(buffer, rgba) when recording=false is a no-op (frameCount unchanged)
    - pushFrame wraps writeIndex at capacity (circular overwrite)
    - After pushing 35 frames into a 30-capacity buffer, frameCount is still 30 and only the last 30 frames are retrievable
    - getOrderedFrames(buffer) returns frames in chronological order (oldest first)
    - getOrderedFrames on a partially-filled buffer (e.g., 10 frames pushed) returns exactly 10 frames
    - getOrderedFrames on a wrapped buffer returns frames in correct order: [writeIndex..capacity, 0..writeIndex]
    - resetBuffer(buffer) sets frameCount=0 and writeIndex=0
    - Setting buffer.recording=false stops frame accumulation; setting back to true resumes
  </behavior>
  <implementation>
    Create src/capture/replay-buffer.ts exporting:
    - ReplayBuffer interface with frames (Uint8ClampedArray | null)[], width, height, writeIndex, frameCount, recording
    - BUFFER_CAPACITY constant = 30 (exported for tests)
    - createReplayBuffer(width: number, height: number): ReplayBuffer
    - pushFrame(buffer: ReplayBuffer, rgba: Uint8ClampedArray): void -- early return if !buffer.recording
    - getOrderedFrames(buffer: ReplayBuffer): Uint8ClampedArray[] -- handles both partial-fill and wrapped cases
    - resetBuffer(buffer: ReplayBuffer): void

    Follow the test pattern from screenshot.test.ts (describe/it blocks, vi mocks where needed).
  </implementation>
</feature>

<feature>
  <name>Feature 2: GIF Encoder Wrapper</name>
  <files>src/capture/gif-encoder.ts, src/capture/__tests__/gif-encoder.test.ts</files>
  <behavior>
    - encodeReplayGif(frames, width, height) returns a Blob with type 'image/gif'
    - The returned Blob has size > 0 (non-empty)
    - GIF header bytes (first 6 bytes) are 'GIF89a' (the animated GIF signature)
    - encodeReplayGif with an empty frames array returns a valid (minimal) GIF blob
    - Each frame is encoded at 100ms delay (10fps playback)
    - GIF is set to loop forever (repeat: 0)
  </behavior>
  <implementation>
    First, install gifenc:
    ```bash
    npm install gifenc
    ```

    Create src/capture/gif-encoder.ts exporting:
    - encodeReplayGif(frames: Uint8ClampedArray[], width: number, height: number): Blob

    Implementation per research pattern:
    - import { GIFEncoder, quantize, applyPalette } from 'gifenc'
    - Create GIFEncoder(), iterate frames, for each: quantize(rgba, 256, { format: 'rgb444' }), applyPalette, writeFrame with delay=100, repeat=0, per-frame palette
    - gif.finish(), return new Blob([gif.bytesView()], { type: 'image/gif' })
    - Handle empty frames array: still call gif.finish() and return valid blob

    For tests: create small synthetic RGBA frames (e.g., 2x2 pixels = 16 bytes of Uint8ClampedArray). Verify blob type, non-zero size, and GIF89a header. Use ArrayBuffer/Uint8Array to inspect header bytes.
  </implementation>
</feature>
