<task type="auto">
  <name>Task 2: Dockerfile, Docker Compose, Caddy config, and dockerignore</name>
  <files>
    Dockerfile,
    docker-compose.yml,
    Caddyfile,
    .dockerignore
  </files>
  <action>
    1. Create `.dockerignore`:
       ```
       node_modules
       dist
       server-dist
       .git
       .planning
       *.local
       .env*
       ```

    2. Create `Dockerfile` (multi-stage):
       - Stage 1 "frontend-build": FROM node:22-bookworm-slim
         - WORKDIR /app
         - COPY package*.json, npm ci
         - COPY tsconfig.json, vite.config.ts, index.html, src/
         - RUN npm run build (produces dist/)
       - Stage 2 "production": FROM node:22-bookworm-slim
         - WORKDIR /app
         - COPY package*.json, RUN npm ci --omit=dev (production deps only including better-sqlite3 native module)
         - COPY tsconfig.server.json, server/
         - RUN npx tsc -p tsconfig.server.json (compiles server TypeScript)
         - COPY --from=frontend-build /app/dist ./dist
         - RUN mkdir -p /app/data (for SQLite volume)
         - ENV PORT=3000, EXPOSE 3000
         - CMD ["node", "server-dist/index.js"]

       CRITICAL: Use node:22-bookworm-slim (NOT Alpine) -- better-sqlite3 requires glibc.

    3. Create `docker-compose.yml`:
       - Service "app": build from ., restart unless-stopped, expose 3000 (internal only), volumes sqlite-data:/app/data, env PORT=3000 and DATABASE_PATH=/app/data/shaken.db
       - Service "caddy": image caddy:2-alpine, restart unless-stopped, ports 80:80 443:443 443:443/udp (HTTP/3), volumes for Caddyfile (read-only), caddy-data:/data, caddy-config:/config, depends_on app
       - Named volumes: sqlite-data, caddy-data, caddy-config

    4. Create `Caddyfile`:
       ```
       shaken.ironhaven.com.au {
           reverse_proxy app:3000
           encode gzip
       }
       ```

    5. Verify Docker build locally (if Docker is available):
       - Run `docker build -t shaken-together .` to confirm the multi-stage build works
       - If Docker is not available locally, verify that `npx tsc -p tsconfig.server.json` compiles without errors and `npm run build` succeeds (both stages represented)
  </action>
  <verify>
    <automated>npx tsc -p tsconfig.server.json && npx tsc --noEmit && npx vitest run</automated>
  </verify>

  </done>
